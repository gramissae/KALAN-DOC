Consultas

En el siguiente tutorial se explica mediante ejemplos las solicitudes que se proponen a desarrollar por lo cual sera nesesario colocar los siguientes codigos de ejemplo.

/* Realizo un ejemplo de solicitud basado en fetch pero es posible usar:
  - Axios
  - HttpxmlRequest
  - JQuery
*/
fetch([RUTA A INGRESAR DE API ], {
  method: [GET, POST, PUT], // Colocar cualquiera de los tipos de consulta
  headers:
 })
.then(res => res.json())
.then(data => console.log(data))
.catch(error => console.errror(error))

Se habilitaran todos los servicios REST en el apartado http://[URL]/service/api/[RUTA]

Requisitos

Solicita a tu Apikey para poder comenzar tu ingreso y poder geneara el token dinamico para cada solicitud via API la cualquiera se te otorgara para agilizar y validad la identidad de cada envio de informacion.

Es nesesario tener un desarrollo basado en las con las siguientes caracteristicas:

  • - JWT - https://jwt.io/
  • - Protocolo de JWT basado en SHA - 256
  • - Clave de intercambio secreta Otrogada por el cliente (Secret para cruzar autentificacion #Archivo Api.js)
  • - Metodo de autentificacion local para con arquitectura opitmizada para ApiRest

Generacion de encabezados

En cada solicitud es necesario para que nuestras Apis convivan una estructura definida en el Header la cual se muestra como ejemplo en GitHub:

Ejemplo en metodos GET, POST, PUT

{
  Content-type: "application/json",
  Accept: "application/json",
  Authorization: {token: [CLIENT-TOKEN], auth: [PROVIDER-TOKEN]} // Opcional agregar un otras credeciales en esta seccion para autentificacion de origen
}

Ejemplo en Login (Obtener Token solo metodo POST)

{
  Content-type: "application/json",
  Accept: "application/json",
  Authorization: {user: [USERNAME], key: [API-KEY]}
}
